# Legacy League

A Club-Penguin-style 2D social hangout game. Sign up, walk a shared map, chat (server-wide + PMs), play mini-games.

## Stack
- **Client**: static vanilla-JS (ES modules), no build step → GitHub Pages.
- **Backend**: Supabase (Auth, Postgres, Realtime). $0 free tier.

See `CLAUDE.md` for architecture + conventions, `docs/` for design.

## Local dev
1. Create a Supabase project (free). Grab the project URL + anon key (Settings → API).
2. Put them in `.env` (repo root):
   ```sh
   SUPABASE_URL=https://YOUR_PROJECT.supabase.co
   SUPABASE_ANON_KEY=YOUR_ANON_KEY
   ```
   (`.env` is gitignored. You can also just `export` these in your shell instead.)
3. Apply DB schema: paste `supabase/setup.sql` into the Supabase SQL Editor and Run.
   Turn off Auth → Providers → Email → "Confirm email" so signup works without an inbox.
4. Run it (generates `config.js` from `.env`, then serves):
   ```sh
   ./scripts/dev.sh
   ```
   Open http://localhost:8000

`config.js` is generated by `scripts/gen-config.sh` — never edit it by hand.

## Deploy
GitHub Actions publishes `public/` to Pages on push to `main`. It regenerates
`config.js` from repo **Secrets** `SUPABASE_URL` + `SUPABASE_ANON_KEY`
(Settings → Secrets and variables → Actions → Secrets). See `.claude/skills/deploy-pages`.

## Security
- Client uses the Supabase **anon key only** (public-safe; Row Level Security protects data).
- Service-role key never touches the client. Every table has RLS.
